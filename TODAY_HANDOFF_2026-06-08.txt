五、后续建议

1. 继续验证步骤3 PDF明细解析

重点看步骤3表格“来源”列：
- 理想情况：大多数或全部显示 PDF。
- 如果很多显示“表3兜底”，说明 PDF 明细行没有被完全识别，需要针对实际 PDF 排版继续增强 parseLimitPdfItems。

2. 重复项目编码风险

当前步骤3主要按项目编码匹配。如果实际工程里存在重复编码，建议升级匹配键：
- 项目编码 + 工程分类
- 或 项目编码 + 项目名称
- 或 项目编码 + 工程量近似匹配

3. 扫描版 PDF 风险

当前 PDF 解析依赖文本可提取。如果最高投标限价 PDF 是扫描图片版，需要接 OCR，否则读不到明细。

4. 步骤6仍不是完整数学优化器

当前材料调价是“逐项差额按资源权重分摊”的实用第一版。若后面要正式用于报价策略，建议继续加：
- 材料单价上下限
- 单条清单最高限价约束
- 不平衡报价策略约束
- 多目标优化或线性规划

5. Git 注意

本工作区因为手动下载/覆盖过仓库，git status 里可能有很多换行符或历史文件差异。提交时要注意不要提交：
